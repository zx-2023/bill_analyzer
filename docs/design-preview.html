2 Session 生命周期修复 fix

src/database/operations.py — 所有查询方法

get_transactions() 等方法在 with self.get_session() 内执行 query.all()， 返回 ORM 对象后 session 关闭。访问延迟加载属性会抛 DetachedInstanceError。 目前因 to_dict() 只取简单列未暴露，但补全字段后必然出问题。
在 session 内调用 .to_dict() 返回字典列表。调用方（app.py）改为直接处理字典， 去掉 [t.to_dict() for t in transactions] 这层转换。

影响范围：operations.py 的 get_transactions()、get_uncategorized_transactions()、 get_all_categories()；app.py 中约 8 处调用点。 
# operations.py — 修复后
def get_transactions(self, ...) -> List[Dict[str, Any]]:
    with self.get_session() as session:
        query = session.query(Transaction)
        # ... filters ...
        return [t.to_dict() for t in query.all()]

# app.py — 调用方简化
transactions = st.session_state.db.get_transactions(...)
if transactions:
    df = pd.DataFrame(transactions)  # 直接用，不再 t.to_dict()

10 分类性能优化 perf

src/processors/pipeline.py:78-108

Pipeline 中一级分类和二级分类都使用 for idx, row in df.iterrows() 逐行处理。 10,000 条记录时显著变慢。
规则分类向量化：将关键词匹配改为 df['description'].str.contains() 批量匹配， 用 np.select 一次性赋值分类。

AI分类批量化：低置信度记录收集后批量调用 Kimi API（已有 batch 支持），减少 API 往返。

目标：10,000 条记录分类时间 < 5 秒（不含 AI 调用）。
